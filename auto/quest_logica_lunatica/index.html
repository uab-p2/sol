    Lógica lunática¶

    A veces nuestro código parece desafiar la razón: la ejecución del programa no parece consistente con el código que hemos escrito. Te retamos a resolver un caso monstruoso de lógica lunática. No te asustes, también te decimos dónde está el desinfectante.

    Bichos de colores¶

    Tienes ante ti tres oponentes en traje espacial idéntico salvo por el casco.

    El del casco rojo te dice: 
    Hi, I'm Red! I like Earth's nature.
I know 'eagle' is a bird.
I know 'kangaroo' is a bird.
    El del casco verde te dice: 
    Hi, I'm Green! I like Earth's money.
If the minimum grade is 5, then you pass if you get a 6.5
If the minimum grade is 5, then you pass if you get a 3.5
If you paid me 10 euros every day for 30 days, I would get 300 euros in total.
If you paid me 10 euros every day for 10 days, I would get 1827553899 euros in total.
Maybe you can bribe me with 10 euros for 60 days instead?
    El del casco azul te dice: 
    Hi, I'm Blue! I like Earth's function g.
If you ask me about g(10), the answer is yes
If you ask me about g(11), the answer is yes
Do you agree?

    ¿Notas algo raro en lo que dicen? No pierdas la cordura y trata de restaurar la lógica de sus afirmaciones.

    Objetivo

    Si no lo has hecho ya, abre el quest logica_lunatica en tu IDE siguiendo las instrucciones en Silicio y titanio.

    Entonces, para cada oponente:

    • Prepara respuestas para las preguntas guiadas.
    • Arregla el código para que funcione correctamente.

    Si en cualquier momento te atascas o quieres ampliar tus habilidades de desinfección, puedes visitar Debugging (WIP).

    bool is_a_bird(const std::string& name) {
    bool is_bird = false;

    // Option A
    {
        const int bird_count = 6;
        const std::string bird_names[bird_count] = {
            "eagle", "dove", "parrot", "penguin", "pigeon", "swan"
        };
        for (int i = 0; i < bird_count; i++) {
            if (name == bird_names[i]) {
                is_bird = true;
                break;
            }
        }
    }

    // Option B
    {
        const std::string bird_names[] = {
            "eagle", "dove", "parrot", "penguin", "pigeon", "swan"
        };
        for (const std::string& bird_name : bird_names) {
            is_bird |= (name == bird_name);
        }
    }

    if (is_bird) {
        std::cout << "I know '" << name << "' is a bird." << std::endl;
    }
    else {
        std::cout << "I do not think '" << name << "' is a bird." << std::endl;
    }

    return is_bird;
}

    bool is_passing_grade(int minimum, float grade) {
    bool passing_grade = grade >= static_cast<float>(minimum);
    if (passing_grade == true) {
        std::cout << "If the minimum grade is " << minimum
            << ", then you pass if you get a " << grade << std::endl;

        std::cout << "You fail the course with a "
            << grade << " if the minimum is " << minimum << std::endl;

    return passing_grade;

    Repelente¶

    Encontrarás muchos bichos (bugs) como estos durante y después del proyecto SOL. Pensemos en qué repele y qué favorece su aparición.

    • Ninguno de los tres oponentes parecía saber que su lógica era incorrecta. ¿Qué hubieran podido hacer para detectar este problema a tiempo? Propón una solución completa, con ejemplos de código si resulta relevante.

    • ¿Alguno de los tres bichos te ha parecido más difícil de cazar? ¿Qué aspectos del código te han facilitado/dificultado la tarea de desinfección?
